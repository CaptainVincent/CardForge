## 2026-06-20 — 規則完備性 · 編輯器一致性 · AI skill

一輪大型迭代:把規則模型補到實務完備、收斂編輯器的互動與版面、加入「官網 → JSON」的 AI skill,並建立測試與 CI。經系統架構、UX/UI、Apple 美學、日本職人四方評估,確認無阻斷問題後收斂發布。

### 規則模型(引擎)
- **上限一般化**:`metric`(回饋金額 / 消費「前 $X」/ 筆數「前 N 筆」)× `window`(單筆 / 單期 / 總);一個回饋可掛多道上限,亦可多規則共用額度池。
- **超額累進**:`tiers.mode = marginal`(各段金額各套費率),有別於 `spend`(整筆取最高符合)。
- **取高 `top` 節點**:依當期累積消費自動取最高 K 類加碼(CUBE 自選 / Custom Cash 風格)。
- **指定特店 `merchant`**:比「類別/MCC」更細的特定商家;可單獨或與類別/通路併用。
- **規則啟用開關 `is_active`**:停用=保留但不參與試算(限時/新戶促銷用,不灌爆日常估算)。
- **備註 `note`**:純文字細則;能結構化的(如「喬山限實體門市」)優先拆規則 + 共用上限池表達。
- 月度上限改為「擇優之後」才套(只算贏家);全路徑(摘要/驗證/推薦/連線規則)一致。

### 編輯器(UX / UI)
- **Inspector 依軸分區**(消費對象 / 支付與通路 / 情境;回饋公式 / 回饋類型 / 進階),統一層級與間距。
- **多來源匯入**:檔案 / 貼上 / 網址;工具列「檔案」更名「資料」。
- **連線可刪**(選取顯示 ×)、**⌘C / ⌘V 跨卡複製貼上**、**⌘D 複製含子樹**。
- 彈出層 Esc / 點外關閉統一;自訂選項納入復原(undo)。
- 設計語言維持日系簡約:漸進式揭露、區塊化,避免一眼過載。

### 範例
- 精煉藝廊為 5 項:綜合 + 取高 / 超額累進 / 多重上限 / 指定特店,各代表模型一個面向。
